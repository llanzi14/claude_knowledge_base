# Tools & Integrations

Newest entries first. Dated `(YYYY-MM-DD)` = date added to this KB.

## Claude Code Built-in Features

- **Dynamic Workflow size setting + `/review` fast single-pass** (2026-07-06) — Workflows expose a size control (small/medium/large) for agent fan-out; `/review` switched to a faster single-pass mode. [Changelog](https://code.claude.com/docs/en/changelog)
- **Transient rate limits auto-retry with backoff** (2026-07-02) — as of v2.1.199, 429 errors unrelated to your actual usage limit are retried automatically instead of failing the turn; `CLAUDE_CODE_RETRY_WATCHDOG` raises the default retry count for non-capacity transient errors to 300. `[ACTION]` Relevant to this KB routine and any other unattended Lanzico automation — should reduce spurious failures. [Changelog](https://code.claude.com/docs/en/changelog)
- **Subagents run in the background by default** (2026-07-01, added to KB 2026-07-03 — missed in the original v2.1.198 entry) — the main session keeps working while a subagent runs, and is notified via the `Notification` hook (`agent_needs_input` / `agent_completed`) rather than blocking until the subagent finishes. [GitHub CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)
- **Claude in Chrome — now GA** (2026-07-01) — as of v2.1.198, Claude's browser control extension is generally available (was preview/limited). `[ACTION]` Evaluate for Lanzico browser-based research/QA workflows that don't need a full Playwright setup. [Changelog](https://code.claude.com/docs/en/changelog)
- **`/dataviz` skill built-in** (2026-07-01) — ships with v2.1.198: guidance and a runnable color-palette validator for building charts, graphs, and dashboards consistently. Already available in this session. `[ACTION]` Use for Lanzico client-facing charts/reports instead of ad hoc styling. [Changelog](https://code.claude.com/docs/en/changelog)
- **Background agents auto-commit/push/open draft PR** (2026-07-01) — as of v2.1.198, background agents started from `claude agents` that finish code work in a worktree now commit, push, and open a draft PR automatically instead of pausing to ask. `[ACTION]` Re-check any Lanzico automation relying on background agents pausing before pushing — behavior has changed. [Changelog](https://code.claude.com/docs/en/changelog)
- **Explore agent uses main session's model (capped at Opus)** (2026-07-01) — previously always ran on Haiku; now inherits the calling session's model for better search quality (higher cost as a tradeoff). [Changelog](https://code.claude.com/docs/en/changelog)
- **`/agents` wizard removed** (2026-07-01) — subagent creation/management is now done by asking Claude directly or editing `.claude/agents/` files; no more interactive wizard. [Changelog](https://code.claude.com/docs/en/changelog)
- **Streaming idle watchdog (on by default)** (2026-07-01) — as of v2.1.196, aborts and retries any provider stream that goes 5 minutes without an event. `CLAUDE_ENABLE_STREAM_WATCHDOG=0` to disable. `[ACTION]` Leave enabled for automated/unattended Lanzico runs — prevents silent hangs. [Changelog](https://code.claude.com/docs/en/changelog)
- **Org default models** (2026-07-01) — admins set a default model in the org console; members see "Org default"/"Role default" in `/model` until they pick their own. [Changelog](https://code.claude.com/docs/en/changelog)
- **`/code-review` — 25% cheaper** (2026-07-01) — v2.1.196 merged five separate cleanup finders into one, cutting token usage roughly 25% per run with no loss of coverage. [Changelog](https://code.claude.com/docs/en/changelog)
- **Artifacts** (2026-06-27) — beta on Team & Enterprise: turn a session's output into a live shareable page on claude.ai that updates in-place as the session works. Replaces the pattern of exporting a report and re-sharing manually. `[ACTION]` Use for Lanzico client-facing deliverables: research reports, changelogs, live status dashboards. [Releasebot](https://releasebot.io/updates/anthropic/claude)
- **`claude mcp login/logout` CLI commands** (2026-06-24) — authenticate and deauthenticate MCP servers directly from the terminal without opening the interactive `/mcp` menu. `--no-browser` flag enables stdin-redirect auth flow for SSH sessions. `[ACTION]` Use for Lanzico MCP servers accessed via SSH or CI environments. [Changelog](https://code.claude.com/docs/en/changelog)
- **`sandbox.credentials` setting** (2026-06-24) — new security control that prevents sandboxed agent commands from reading credential files or secret env vars. Safe default for any automated agentic run touching sensitive environments. `[ACTION]` Enable in all Lanzico CI/automation configs. [Changelog](https://code.claude.com/docs/en/changelog)
- **Remote MCP idle timeout** (2026-06-24) — MCP tool calls now abort after 5 minutes of no response (was: hang indefinitely). Override via `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`. Prevents CI jobs from stalling silently. [Changelog](https://code.claude.com/docs/en/changelog)
- **`!` bash auto-respond** (2026-06-24) — `!<command>` now triggers Claude to respond to the shell output automatically; set `"respondToBashCommands": false` in settings.json to preserve previous context-only behavior. [Changelog](https://code.claude.com/docs/en/changelog)
- **Skills in nested directories** (2026-06-22) — as of v2.1.178, skills placed in nested `.claude/skills/` subdirectories load locally without `--global` flag. Enables per-feature or per-service skill scoping in monorepos. `[ACTION]` Restructure Lanzico skills into per-service subdirectories if needed. [Changelog](https://code.claude.com/docs/en/changelog)
- **Agent teams via direct Agent tool** (2026-06-22) — `TeamCreate`/`TeamDelete` removed in v2.1.178; teams are now implicit — spawn subagents directly with the Agent tool. Simpler orchestration model. [Changelog](https://code.claude.com/docs/en/changelog)
- **`/config key=value` shorthand** (2026-06-22) — change any config setting from within a session prompt (e.g., `/config thinking=false`, `/config model=fable-5`). [Changelog](https://code.claude.com/docs/en/changelog)
- **`attribution.sessionUrl` setting** (2026-06-22) — set to `false` to omit the claude.ai session link from git commits and PRs. `[ACTION]` Apply if Lanzico commits go to client-visible repos. [Changelog](https://code.claude.com/docs/en/changelog)
- **`CLAUDE_CLIENT_PRESENCE_FILE`** (2026-06-22) — env var to suppress mobile push notifications during automated/background runs. Useful in CI. [Changelog](https://code.claude.com/docs/en/changelog)
- **`enforceAvailableModels`** (2026-06-22) — managed setting to prevent users from switching outside an approved model list. Useful for cost governance in teams. [Changelog](https://code.claude.com/docs/en/changelog)
- **Plugin marketplace search** (2026-06-12) — search bar in `/plugin` for browsing community plugins. [Changelog](https://code.claude.com/docs/en/changelog)

## Slack / Team Collaboration

- **Claude Tag** (2026-06-25) — Anthropic's Slack-native persistent AI teammate (Enterprise & Team plans). Admins add Claude to channels with scoped access (specific channels, tools, approved data sources). Ambient mode lets Claude monitor channels and post proactively without @mention. One shared identity per channel — the whole team can continue conversations. Each instance stays channel-scoped; a legal-channel Claude cannot reach engineering channels. Token spend ceiling configurable per channel. `[ACTION]` Add Claude Tag to Lanzico's Slack via Enterprise/Team admin panel; start with one internal-ops or client-delivery channel. [Anthropic](https://www.anthropic.com/news/introducing-claude-tag) / [Fortune](https://fortune.com/2026/06/23/anthropic-claude-tag-virtual-employee-tool-slack/) / [ComputerWorld](https://www.computerworld.com/article/4189055/anthropics-claude-tag-aims-to-turn-workplace-ai-from-a-personal-assistant-into-a-teammate.html)
