## Claude Code

- **v2.1.207** (2026-07-11, latest) — **Security fix**: remote managed settings applied from a non-interactive run (`claude -p`, the SDK) were being permanently recorded as "consented" **without ever showing the security consent dialog** — relevant to any headless/scheduled Claude Code automation, including this KB routine. **Security fix**: plugin hooks/monitors/MCP `headersHelper` no longer allow `${user_config.*}` interpolation in shell-form commands (a shell-injection vector) — hooks must now use exec form (`args` array) or `$CLAUDE_PLUGIN_OPTION_<KEY>`. Also fixed spurious prompt-injection warnings triggered by benign system-generated conversation updates. **Config change**: auto mode no longer reads `autoMode` from repo-resident `.claude/settings.local.json` — use `~/.claude/settings.json` instead; if that's how auto mode was configured anywhere at Lanzico, it silently stopped applying. Auto mode is now available on Bedrock/Vertex/Foundry without the `CLAUDE_CODE_ENABLE_AUTO_MODE` opt-in. Bedrock/Vertex/Claude Platform on AWS now default to Opus 4.8. Also: fixed terminal freeze/keystroke lag while streaming very long lists/tables/code blocks; fixed the auto-updater silently overwriting a custom launcher script/symlink at `~/.local/bin/claude` (`/doctor` now flags an externally managed launcher); fixed a crash loop in agent teams from a malformed teammate mailbox message; several Remote Control, worktree, and Deep Research display fixes; `/usage-credits` now rejects malformed pasted amounts instead of silently truncating them. `[ACTION]` See Knowledge Base Updates below.
