## Claude Tag — New Product (June 23, missed from previous digest)

- **Claude Tag launched** — Anthropic's new Slack-native AI teammate, available now for Enterprise and Team customers. Claude joins shared Slack channels as a persistent team member (not a per-user bot). [Anthropic](https://www.anthropic.com/news/introducing-claude-tag) / [TechCrunch](https://techcrunch.com/2026/06/23/anthropics-claude-tag-is-learning-your-company-one-slack-message-at-a-time/)
- **Shared identity** — one Claude per channel; the whole team can @mention it, see its work, and hand off mid-conversation. Builds persistent context from channel history and approved sources.
- **Ambient mode** — when enabled, Claude monitors the channel proactively: flags relevant information, follows up on dropped threads, posts updates without being @mentioned.
- **Admin controls** — admins set which channels, tools, and data sources each Claude Tag instance can access; each instance is channel-scoped (legal Claude can't see engineering channels). Token spend ceiling per channel for cost governance.
- **Internal signal** — Claude Code team has used Claude Tag internally all year; it now writes 65% of the product team's code, including most of the Claude Tag product itself. [Fortune](https://fortune.com/2026/06/23/anthropic-claude-tag-virtual-employee-tool-slack/)

## Community Best Practices (newly surfaced)

- **`/compact` at 50% context** — community consensus on r/ClaudeCode: trigger `/compact` manually when context reaches 50% (not the default auto-trigger near 70%); this prevents the "agent dumb zone" where performance degrades. [DEV.to](https://dev.to/galian/claude-code-workflow-best-practices-that-ship-code-na)
