## Claude Code

- **v2.1.200** (July 3) — the permission mode previously shown as "default" is now labeled **"Manual"** across the CLI, `claude --help`, VS Code, and JetBrains (config value is still `default`; `manual` is accepted as an alias, e.g. `--permission-mode manual`). Functionally this appears to be a naming/clarity change rather than new restrictions — "Manual" already meant "review every action." More relevant: **`AskUserQuestion` dialogs no longer auto-continue by default** — previously an idle dialog would eventually auto-continue; now it waits indefinitely unless you opt into an idle timeout via `/config`. `[ACTION]` For unattended/scheduled Claude Code runs (including this routine), verify whether any prompt paths could trigger `AskUserQuestion` and, if so, set an idle timeout via `/config` — otherwise a scheduled run could hang waiting for input that will never come. [Changelog](https://code.claude.com/docs/en/changelog) / [X thread](https://x.com/ClaudeCodeLog/status/2073091434123591872) / [Permission modes doc](https://code.claude.com/docs/en/permission-modes)
- Other v2.1.200 fixes: crash at startup when `disabledMcpServers`/`enabledMcpServers` in `.claude.json` is a non-array value; background sessions no longer silently stop mid-turn after sleep/wake or when reopening a stalled session; JetBrains terminal flickering fixed on 2026.1+; subagents and context compaction inherit the session's extended-thinking config; syntax highlighting upgraded to highlight.js 11 for better accuracy in code blocks/diffs/previews. [Changelog](https://code.claude.com/docs/en/changelog)
