# Digest — 2026-07-09

Three new Claude Code releases since the last digest (v2.1.202 → v2.1.203 → v2.1.204 → v2.1.205), mostly reliability/UX fixes plus a couple of items directly relevant to this routine's own automation. Fable 5's free/included access window was quietly extended by 5 days. No new community discussion surfaced.

## Claude Code

- **v2.1.203** (2026-07-07) — login-expiry warning before background sessions get interrupted; grey ⏸ badge for manual permission mode; session's additional working directories now exposed via MCP `roots/list`. Large batch of background-agent/macOS/Windows reliability fixes (stale daemon token, stale `PATH`, worktree isolation bugs, crash-looping on deleted working dirs). ~7MB cut from binary size and startup memory. [Changelog](https://code.claude.com/docs/en/changelog)
- **v2.1.204** (2026-07-08) — fixed hook events not streaming during `SessionStart` hooks in headless sessions, which could cause remote/background workers to be **idle-reaped mid-hook**. `[ACTION]` Directly relevant to this KB routine and any other scheduled/headless Claude Code run using a `SessionStart` hook — update to pick up the fix if idle-reaping during startup hooks has ever been an issue. [Changelog](https://code.claude.com/docs/en/changelog)
- **v2.1.205** (2026-07-08, latest) — `/doctor` is now a full setup checkup (`/checkup` alias); auto mode now blocks tampering with session transcript files and asks before running `rm -rf` on an unresolvable variable; **background task notifications now explicitly state that no human input has occurred, preventing fabricated in-transcript approvals from being acted on**. Also: `claude mcp add-from-claude-desktop` no longer stalls on bad server names, auto-update memory usage cut ~400MB, agent-view rows now link PRs they touch. [Changelog](https://code.claude.com/docs/en/changelog)
