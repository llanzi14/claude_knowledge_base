# Digest — 2026-07-17

- **Claude Code v2.1.212 released** (2026-07-17) — headline items are two new self-protection limits directly relevant to this KB routine and any other agentic/scheduled automation: a **session-wide WebSearch cap** (default 200 calls, tunable via `CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`) and a **per-session subagent-spawn cap** (default 200, tunable via `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`, resets on `/clear`) — both guard against runaway search/delegation loops. MCP tool calls running longer than 2 minutes now auto-background so the session doesn't stall (`CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS` to tune/disable). **Security fix**: plan mode was auto-running file-modifying Bash commands (e.g. `touch`, `rm`) **without a permission prompt or SDK `canUseTool` callback** — now fixed, plan mode should never execute file-modifying commands unapproved. `/fork` now copies a conversation into its own background session row (the old in-session-subagent behavior is now `/subtask`). New `claude auto-mode reset` command. Also: fixed a `continue:false` hook's halt being silently dropped when a tool fails mid-stream; fixed hook infrastructure errors being misreported as user rejections (same theme as earlier `[ACTION]` items in this KB about unattended runs stalling on misreported rejections); fixed SIGTERM during a Bash tool orphaning the process tree in print/SDK mode; `Enterprise forceLoginMethod` now enforced for VS Code, SDK, `setup-token`, and `install-github-app` logins (previously terminal-only); reasoning effort level now recorded per assistant message in transcripts. `[ACTION]` Update to v2.1.212 — the plan-mode Bash-permission bypass is a real security gap worth closing promptly; separately, note the new WebSearch/subagent caps (200 each) in case any Lanzico Workflow-based automation fans out that wide. [Changelog](https://code.claude.com/docs/en/changelog)
- **Claude Code v2.1.211 released** (2026-07-15, not caught in the last digest) — **Security fix**: permission previews relayed to chat channels (e.g. Slack approvals) now neutralize bidirectional-override, zero-width, and look-alike quote characters, so a malicious tool input can no longer visually alter what the approval message appears to say. **Security fix**: auto mode was overriding a hook's explicit `ask` decision for unsandboxed Bash — a hook `ask` now correctly floors the decision at a prompt instead of being silently auto-approved. Fixed a prompt-caching billing regression on **Bedrock, Vertex, Mantle, and Foundry** that was billing the trailing system-context block as fresh input tokens on every request (cost-relevant if Lanzico runs Claude Code through any of those gateways). Improved background-agent result reporting so Claude **reports the real status of still-running agents and waits for actual completion instead of fabricating results** — directly reinforces the "no fabricated approvals/results" hardening theme already logged in this KB. Added `--forward-subagent-text` / `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` to stream subagent text/thinking in `stream-json` output (useful for observability pipelines). `[ACTION]` Update to v2.1.211+ — the chat-approval spoofing fix and the hook `ask`-override fix both matter for any Lanzico setup that approves tool calls via a chat channel or relies on hooks to gate unsandboxed Bash. [Changelog](https://code.claude.com/docs/en/changelog)
