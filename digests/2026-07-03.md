## Claude Code

- **v2.1.199** (July 2) — mostly reliability and retry fixes. Notable: **transient 429 rate-limit errors now auto-retry with backoff for subscribers** instead of failing the turn, and `CLAUDE_CODE_RETRY_WATCHDOG` raises the default retry count for non-capacity transient errors to 300 (previously capped at 15 via `CLAUDE_CODE_MAX_RETRIES`) — directly relevant to unattended/scheduled runs like this one. Also fixes: subagents cut off by a rate limit or server error now return partial work instead of silently failing; subagents that hit an API error (e.g. usage limit) now correctly report failure instead of a false "success"; streaming responses are kept (with an incomplete-response notice) instead of discarded when a mid-stream server error occurs; Linux background-agent daemon no longer kills itself and every running agent every ~50s after an unclean shutdown; background agents no longer fail to cold-start over SSH on macOS (regression in 2.1.196); `claude stop` no longer gets silently undone by a racing background-agent respawn; idle subagents that vanished from the agent panel now collapse into an expandable row; stacked slash-skill invocations (`/skill-a /skill-b do X`) now load all leading skills (up to 5), not just the first. [Changelog](https://code.claude.com/docs/en/changelog)
- **Correction to the July 1 entry (v2.1.198)** — the KB previously didn't capture that v2.1.198 also made **subagents run in the background by default** (Claude keeps working while a subagent runs and is notified via the `Notification` hook — `agent_needs_input` / `agent_completed` — when it needs input or finishes), a gradual rollout that's now fully on. This explains the behavior change some sessions may have already noticed. [GitHub CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)
