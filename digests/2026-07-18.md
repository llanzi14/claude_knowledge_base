# Digest — 2026-07-18

- **Claude Code v2.1.214 released** (2026-07-18) — security-heavy batch of Bash/permission-bypass fixes: single-segment `dir/**` allow rules (e.g. `Edit(src/**)`) were wrongly auto-approving writes to any nested `dir/` anywhere in the tree, not just under cwd; a Windows PowerShell 5.1 permission-check bypass; Bash permission checks now fail closed on redirect forms the analyzer parsed differently than bash itself, on commands over 10,000 characters (now always prompt), and on zsh variable subscripts/modifiers inside `[[ ]]` that were being treated as inert text; `help`/`man` commands with unsafe options/substitutions no longer auto-approve; permission prompts on remote sessions could previously proceed before the local confirmation dialog completed. New **`EndConversation` tool** — Claude can now end a session with a highly abusive user or jailbreak attempt, mirroring claude.ai behavior since 2025. Also directly relevant to this KB routine itself: **fixed scheduled tasks refusing their own configured prompt as untrusted input — the fired prompt is now delivered as the session's assigned task** (this exact framing shows up in this session's own system context). Large batch of background-session/daemon fixes (idle sessions no longer stay alive forever, completed sessions removable via `claude rm`, stopped-session restore bug), Windows PowerShell-tool encoding/hang fixes, OTel logging enhancements (`message.uuid`, `client_request_id`, `tool_source`), and a `docker`/Podman remote-mode permission-prompt gap closed. `[ACTION]` Update to v2.1.214 — several real permission-bypass fixes here (the `dir/**` scoping bug is the most consequential: it means broad-looking allow rules were quietly wider than intended).
