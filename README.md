# Claude Knowledge Base

An automatically maintained knowledge base on Claude and Claude Code: new features, best practices, tools, and community insights. Updated daily at 07:00 by a scheduled routine.

## Purpose

Stay informed on everything that improves how Claude is used at Lanzico and other projects, without manually tracking docs, changelogs, and community channels.

## Structure

| Path | Contents |
|---|---|
| `ROUTINE.md` | Instructions the daily routine follows (scope, token budget, update rules) |
| `SOURCES.md` | The list of sources scanned each run |
| `knowledge-base/releases-and-features.md` | Notable releases, changelog highlights, new features |
| `knowledge-base/best-practices.md` | Workflows, prompting patterns, configuration tips |
| `knowledge-base/tools-and-integrations.md` | MCP servers, SDKs, skills, plugins, third-party tools |
| `knowledge-base/community-insights.md` | Useful findings from Reddit, YouTube, social media |
| `digests/YYYY-MM-DD.md` | One short digest per run: what was new that day, with links |

## How to use it

- **Quick catch-up:** read the latest file in `digests/`.
- **Reference:** the `knowledge-base/` files are cumulative and curated — newest entries at the top, stale entries pruned.
- **Actionable items for Lanzico** are tagged with `[ACTION]` inside knowledge-base files so they're easy to grep.
